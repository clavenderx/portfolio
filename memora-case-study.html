Overview

Designing cognitive training for an aging population

As part of IAT 351: Advanced Human-Computer Interaction, our team developed Memora‚Äîa mobile app that makes cognitive exercise accessible for elderly users through thoughtful, research-backed design.

The Problem

  • Cognitive decline affects memory, attention, and daily functioning in elderly adults
  • Most cognitive training apps use interaction patterns designed for younger users
  • Complex gestures and small UI elements create barriers to access

The Opportunity

  • Design specifically for age-related changes in vision, motor control, and cognition
  • Apply evidence-based exercises proven to support cognitive health
  • Create an intuitive, dignified experience that respects elderly users

Research & Discovery

Understanding elderly interaction patterns

I led research into age-appropriate interaction design, examining how physical and cognitive changes impact mobile app usage. This research directly informed our design decisions.

Icon Legibility

Elderly users need larger icons (minimum 44√ó44pt) with high contrast ratios. Visual acuity decreases with age, making small or low-contrast elements difficult to distinguish.

üß†

Cognitive Load

Reduce mental effort by minimizing steps, using clear labels, and avoiding nested navigation. Working memory capacity decreases, making complex flows frustrating.

üëÜ

Tap Over Gestures

Direct tapping is more reliable than swipes or pinches. Fine motor control declines with age, and multi-touch gestures can cause accidental inputs or confusion.

Evidence-Based Exercise Selection

I evaluated cognitive training literature to identify exercises targeting specific cognitive domains:

Memory exercises using recognition and recall tasks with familiar objects

Attention games requiring sustained focus and pattern identification

Logic challenges involving simple problem-solving and sequencing

Daily practice tracking to encourage consistent cognitive engagement

Design Process

From research insights to interface design

Using our research findings, I established design principles and created prototypes in Figma, iterating based on accessibility guidelines and team feedback.

1. Large Touch Targets

All interactive elements are minimum 56√ó56pt with generous spacing to prevent mis-taps and accommodate reduced motor precision.

2. High Contrast & Readability

WCAG AAA contrast ratios (7:1) for text, with font sizes starting at 18pt. Clear visual hierarchy through size and color, not subtlety.

3. Simple, Direct Navigation

Flat information architecture with clear back buttons. No hidden menus, hamburger icons, or gesture-based navigation.

4. Familiar Patterns

Card-based layouts with clear labels and icons. Visual feedback for all interactions to confirm actions were registered.

Key Design Decisions

Through prototyping, we refined several critical interactions:

Color-coded game categories help users quickly identify different exercise types without reading labels.

Positive reinforcement language ("You've completed 3 exercises!") rather than streak counters that could feel punitive if missed.

Progress visualization using simple trophy icons rather than graphs or charts that require interpretation.

Development

Translating design to React Native

I implemented the final designs in React Native, ensuring our accessibility decisions carried through to the functional app. Technical choices prioritized performance and native platform patterns.

Technical Implementation

Component architecture: Built reusable GameCard, Button, and Header components with accessibility props built in.

Touch handling: Used React Native's Pressable with generous hitSlop values to expand touch areas beyond visual bounds.

Accessibility features: Implemented screen reader support with semantic labels and state announcements for all interactive elements.

// Example: Accessible game card component <Pressable accessible={true} accessibilityLabel={`${title} game`} accessibilityHint="Double tap to start exercise" hitSlop={{top: 10, bottom: 10, left: 10, right: 10}} style={styles.gameCard} onPress={() => navigation.navigate('Game', {type})} > <Icon name={iconName} size={48} /> <Text style={styles.largeText}>{title}</Text> </Pressable>

Challenges & Solutions

Challenge: Ensuring consistent tap target sizes across different device sizes.

Solution: Used percentage-based dimensions with minimum pixel values to scale appropriately while maintaining accessibility standards.

Challenge: Balancing visual appeal with simplicity for our demographic.

Solution: Subtle animations (200ms fades) for feedback without overwhelming users. Disabled parallax and complex transitions.

Reflection

Learnings & next steps

What I Learned

  • Research-driven design creates more effective solutions than assumptions
  • Accessibility constraints can actually improve design for all users
  • Translating Figma to code requires constant accessibility checks
  • Designing for aging users requires empathy and restraint

Challenges Overcome

  • Balancing visual interest with simplicity for our demographic
  • Choosing exercises with solid research backing
  • Maintaining design consistency across multiple developers
  • Testing accessibility without direct access to elderly users

Future Improvements

If I could continue this project, I would:

Conduct usability testing with actual elderly users to validate our design decisions and identify pain points we might have missed.

Add difficulty scaling that adapts to individual performance without being obvious or discouraging.

Implement voice interactions as an alternative input method for users with limited mobility.

Build social features carefully‚Äîallowing family members to check progress without creating surveillance anxiety.
